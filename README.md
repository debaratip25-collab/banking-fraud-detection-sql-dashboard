# 🏦 Banking Transactions & Fraud Detection Dashboard

## 📌 Overview

This project simulates a real-world banking system that tracks customer transactions and detects suspicious activity using SQL-based fraud detection rules. It is deployed using Docker with MySQL as the backend and visualized through an interactive Python dashboard built with Streamlit.

---

## 🎯 Objective

* Design a normalized banking database
* Simulate realistic transaction data
* Detect fraudulent patterns using SQL logic
* Build an interactive dashboard for analysis

---

## 🛠️ Tech Stack

* **Database:** MySQL (Dockerized)
* **Backend Logic:** SQL (DDL, DML, Views, Triggers)
* **Visualization:** Python (Streamlit + Plotly)
* **Tools:** Docker, VS Code

---

## 🧱 Database Design

The system follows a normalized relational structure:

* **Customers** – Customer details and KYC status
* **Accounts** – Linked bank accounts
* **Transactions** – All financial activity
* **Branches** – Bank branch details
* **Risk Events** – Fraud flags generated by rules

---

## 🚨 Fraud Detection Logic

Implemented rule-based fraud detection using SQL:

* High-value transactions above threshold
* Multiple transactions in a short period (velocity)
* Suspicious transaction patterns
* Automated flagging using triggers

---

## 📊 Dashboard Features

* Daily transaction trends
* Channel-wise transaction analysis
* Fraud alert distribution
* Top risky accounts
* Key KPIs (Transactions, Alerts, Accounts)

---

## 📈 Key Insights

* A significant portion of transactions fall under small-value categories, with occasional spikes indicating potential fraud
* Certain channels (e.g., UPI, POS) dominate transaction volume
* High-risk accounts can be identified through repeated flagged events
* Fraud detection rules help reduce manual monitoring effort

---

## ⚙️ How to Run

### 1. Start Database (Docker)

```bash
docker-compose up -d
```

### 2. Run Dashboard

```bash
pip install -r requirements.txt
streamlit run app.py
```
